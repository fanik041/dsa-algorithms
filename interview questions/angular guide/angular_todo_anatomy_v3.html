🏛️
AppModule
app.module.ts · @NgModule
@NgModule
🏛️ City Hall — Before any building opens in the city, it must register at City Hall. City Hall keeps a list of every building (declarations), every road and utility (imports), and every delivery company (providers). Without registration, Angular says "I don't know what that is."
AppModule is the root NgModule. Angular reads it at startup to wire the whole app. Angular 17+ apps skip this in favour of standalone components + app.config.ts.
@NgModule({ declarations: [AppComponent, TodoListComponent ...], imports: [BrowserModule, RouterModule, HttpClientModule], providers: [TodoService], bootstrap: [AppComponent] })
🏠
Component
TypeScript class + HTML template
@Component
🏠 A Building — Every component has two floors: the lobby (HTML template — what visitors see) and the manager's office (TypeScript class — the brain). The manager stores data, calls services, and reacts to visitor actions.
A component = @Component decorator + TypeScript class. The decorator links it to its template and stylesheet. The class holds state (properties) and logic (methods).
@Component({ selector: 'app-todo-list', // HTML tag name templateUrl: './todo-list.html', // the lobby styleUrls: ['./todo-list.css'] }) export class TodoListComponent implements OnInit { ... }
🏪
TodoService
todo.service.ts · @Injectable
@Injectable
🏪 The Delivery Company — Buildings don't go shopping themselves. They call the delivery company. The service goes to the API (the supermarket), fetches the data, and delivers it back via a pipe (Observable). One service, shared by all components.
Services contain all HTTP calls and business logic. Components stay "thin" — they just call the service and display the result. Click Scene 4 for the full service code.
💉
Dependency Injection
Angular's automatic wiring
💉 The City Directory — Instead of a building going out to find a delivery company, you write in the constructor: "I need a TodoService." Angular looks in its directory, finds the one company that already exists, and hands it to you. You never write new TodoService().
DI = declare the dependency in the constructor, Angular provides it. providedIn: 'root' makes the service a singleton shared by the whole app.
// Angular injects TodoService automatically constructor(private todoService: TodoService) {} // Now this.todoService is ready to use!
🌊
RxJS Observables
Reactive data streams
🌊 Water Pipes — An Observable is a pipe. Nothing flows until you turn on the tap (subscribe). You can add filters mid-pipe: map changes the shape of the water, catchError handles leaks. The | async pipe in HTML is a self-sealing tap — it closes automatically when the building is demolished.
HTTP calls in Angular return Observables. .pipe() chains operators. Use the async pipe in templates to avoid manual subscription management.
todos$ = this.todoService.getAll(); // just assign // In HTML — auto subscribes AND auto unsubscribes: <li *ngFor="let t of todos$ | async">{{ t.title }}</li>
🗺️
Router
app-routing.module.ts
🗺️ The Road System — The router watches the URL. Type / → drives you to TodoListComponent. Type /todos/5 → drives you to TodoDetailComponent showing todo #5. The <router-outlet> is the magic door where the matched building appears.
Angular Router = client-side navigation. No full page reloads. Routes config maps URL paths to components. <router-outlet> is the render target.
const routes = [ { path: '', component: TodoListComponent }, { path: 'todos/:id', component: TodoDetailComponent } ];

HTTP API
JSONPlaceholder — mock REST API
🌐 The Supermarket — The API is outside the city. The service drives out, picks up the data (JSON), and brings it back via an Observable. We use JSONPlaceholder as a free fake API that returns realistic todo data.

📋
@NgModule
The Registration Form
📋 Before the city opens, the mayor fills out one registration form. The form has four sections: which buildings exist, what utilities are available, which delivery companies are approved, and which building opens first.
@NgModule is a TypeScript decorator that adds metadata to a class. Angular reads this metadata at startup to understand your application's structure.
🏠
declarations[]
Which buildings exist
🏠 Building Permits — Every component, directive, and pipe must be declared here. If it's not on the list, Angular won't recognise the HTML tag and will throw an error like "unknown element." Each item can only be declared in ONE module.
declarations[] lists all components, directives, and pipes belonging to this module. Angular uses this to resolve component selectors in templates.
📦
imports[]
Utilities the city needs
📦 Utilities — The city doesn't build its own electricity. BrowserModule = browser DOM support. AppRoutingModule = the road system. HttpClientModule = ability to call APIs. FormsModule = two-way binding with [(ngModel)].
imports[] brings in other NgModules whose exported declarations, directives, and pipes become available in this module's templates.
💉
providers[]
Approved services
💉 Approved Vendors — Listing a service in providers[] tells the DI system it's approved to operate in this city. Any component can then request it in its constructor. Modern Angular uses providedIn: 'root' directly in the service instead — no need to list it here at all.
// Modern preferred way — in the service itself: @Injectable({ providedIn: 'root' }) export class TodoService { ... } // Now you don't need to add it to providers[]
🚀
bootstrap[]
The first building to open
🚀 Grand Opening — When Angular starts, it opens exactly one building first: AppComponent. It looks for <app-root> in index.html and renders AppComponent there. Everything else loads inside it.
<!-- index.html --> <body> <app-root></app-root> <!-- Angular renders AppComponent here --> </body>

Standalone (Angular 17+)
No NgModule needed
⚡ Self-Permitted Buildings — In Angular 17+, components can register themselves with standalone: true. No City Hall needed. The component lists its own imports. app.config.ts replaces AppModule as the startup config.

import { Component, OnInit, OnDestroy } from '@angular/core';
import { Router } from '@angular/router';
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';
import { TodoService } from '../services/todo.service';
import { Todo } from '../models/todo.model';

@Component({
  selector:    'app-todo-list',
  templateUrl: './todo-list.component.html',
  styleUrls:   ['./todo-list.component.css']
})
export class TodoListComponent implements OnInit, OnDestroy {

  // ── State (manager's notepad) ─────────────────────
  todos:        Todo[]   = [];
  newTodoTitle: string  = '';  ← two-way bound to input
  isLoading:    boolean = true;
  errorMsg:     string  = '';

  // ── Unsubscribe helper ────────────────────────────
  private destroy$ = new Subject<void>();

  // ── DI: Angular injects these automatically ───────
  constructor(
    private todoService: TodoService,
    private router:      Router
  ) {}

  // ── Lifecycle: runs once after component mounts ───
  ngOnInit(): void {
    this.todoService.getAll()
      .pipe(takeUntil(this.destroy$))  // auto-unsubscribe
      .subscribe({
        next:  (todos) => {
          this.todos     = todos;
          this.isLoading = false;
        },
        error: (err) => {
          this.errorMsg  = 'Failed to load todos';
          this.isLoading = false;
        }
      });
  }

  // ── Add a new todo ────────────────────────────────
  addTodo(): void {
    const title = this.newTodoTitle.trim();
    if (!title) return;

    this.todoService.add(title)
      .pipe(takeUntil(this.destroy$))
      .subscribe(newTodo => {
        this.todos     = [...this.todos, newTodo];
        this.newTodoTitle = '';  // clears the input
      });
  }

  // ── Delete a todo ─────────────────────────────────
  deleteTodo(id: number): void {
    this.todoService.delete(id)
      .pipe(takeUntil(this.destroy$))
      .subscribe(() => {
        this.todos = this.todos.filter(t => t.id !== id);
      });
  }

  // ── Toggle complete ───────────────────────────────
  toggleComplete(todo: Todo): void {
    const updated = { ...todo, completed: !todo.completed };
    this.todoService.update(updated)
      .subscribe(t => {
        this.todos = this.todos.map(x => x.id === t.id ? t : x);
      });
  }

  // ── Navigate to detail page ───────────────────────
  viewDetail(id: number): void {
    this.router.navigate(['/todos', id]);
  }

  // ── Cleanup on component destroy ──────────────────
  ngOnDestroy(): void {
    this.destroy$.next();
    this.destroy$.complete();
  }
}

🏷️
@Component decorator
Registers the component with Angular
🏷️ The Name Badge — The decorator is like a name badge. It tells Angular: "my HTML tag is app-todo-list, my lobby is in this file, my stylesheet is in that file." Without it, the class is just a plain TypeScript class Angular knows nothing about.
@Component({ selector: 'app-todo-list', // <app-todo-list> in HTML templateUrl: './todo-list.component.html', styleUrls: ['./todo-list.component.css'] })
💉
constructor — DI
Angular auto-provides services
💉 Room Service — The constructor is where the manager lists what they need. They write "I need a TodoService and a Router." Angular's front desk looks up the directory and hands them both automatically. You never write new TodoService().
Constructor injection: declare parameters with types, Angular provides them. The private keyword also creates a class property automatically.
constructor( private todoService: TodoService, // ← Angular provides this private router: Router // ← and this ) {} // this.todoService is now ready to use anywhere in the class
🌊
ngOnInit + subscribe
Turn on the tap when building opens
🌊 Opening Day — ngOnInit runs once, right after the building opens. It turns on the tap: calls getAll() (starts HTTP request), and the data flows in as an Observable. takeUntil(destroy$) is the automatic valve that closes the tap when the building closes (component destroyed) — preventing memory leaks.
ngOnInit is Angular's "after construction" hook. subscribe() starts the Observable. takeUntil + Subject is the clean unsubscribe pattern.
private destroy$ = new Subject<void>(); ngOnInit() { this.todoService.getAll() .pipe(takeUntil(this.destroy$)) // auto-close tap .subscribe({ next: todos => this.todos = todos }); } ngOnDestroy() { this.destroy$.next(); // emit = close all taps this.destroy$.complete(); // clean up the signal }
➕
addTodo() / toggleComplete()
Event handler methods
🔔 When the Doorbell Rings — These methods run when the user clicks a button or presses Enter. They call the service (delivery company) to save the change to the API, then update the local todos array so the screen refreshes.
Event handler methods are called by event bindings in the template. After the service responds, update state immutably (spread operator) so Angular's change detection picks up the change.
addTodo(): void { const title = this.newTodoTitle.trim(); if (!title) return; // guard: no empty todos this.todoService.add(title) .subscribe(newTodo => { this.todos = [...this.todos, newTodo]; // immutable update this.newTodoTitle = ''; // clear input }); }
🗑️
deleteTodo(id)
Remove from API + update local array
🗑️ Demolishing a Room — When the user clicks delete, the service sends a DELETE request to the API. When it succeeds, the local array is filtered to remove the item. Angular notices the new array reference and re-renders the list.
deleteTodo(id: number): void { this.todoService.delete(id).subscribe(() => { // filter creates a NEW array — Angular re-renders this.todos = this.todos.filter(t => t.id !== id); }); }
📻
[(ngModel)] Two-way binding
The walkie-talkie
⇄ both ways
📻 Walkie-Talkie — The input box and newTodoTitle are in constant two-way contact. User types → variable updates instantly. Manager sets variable to '' → input clears instantly. Both sides always in sync. Requires FormsModule to be imported.
<input [(ngModel)]="newTodoTitle" /> <!-- Same as writing both directions manually: --> <input [ngModel]="newTodoTitle" <!-- class→HTML --> (ngModelChange)="newTodoTitle=$event" <!-- HTML→class --> />
📺
{{ }} and [prop] One-way binding
The TV screen
→ only
📺 The TV Screen — The class broadcasts to the template. {{ todo.title }} displays text. [disabled]="!title" sets an attribute. [class.completed]="todo.completed" adds/removes a CSS class. It's one-way: class→template only.
<span>{{ todo.title }}</span> <!-- interpolation --> <button [disabled]="!title">Add</button> <!-- property --> <li [class.done]="todo.completed"> <!-- CSS class --> <img [src]="todo.imageUrl" /> <!-- attribute -->
📑
*ngFor
The photocopier
📑 The Photocopier — *ngFor takes one <li> template and copies it once for every item in the array. 10 todos = 10 copies, each filled with different data. trackBy is like labelling each photocopy with an ID so Angular only reprints changed copies.
<li *ngFor="let todo of todos; trackBy: trackById"> {{ todo.title }} </li> // In the class: trackById(index: number, todo: Todo): number { return todo.id; // Angular uses this to avoid full re-renders }

*ngIf
The light switch
💡 The Light Switch — *ngIf doesn't just hide elements — it removes them from the DOM entirely. When isLoading is true, the spinner is in the DOM and the list is not. When isLoading is false, it flips: spinner gone, list appears.
<div *ngIf="isLoading">⏳ Loading...</div> <!-- Angular 17+ native @if (no import needed): --> @if (isLoading) { <div>⏳ Loading...</div> } @else { <ul>...</ul> }

todo.service.ts
import { Injectable }  from '@angular/core';
import { HttpClient }   from '@angular/common/http';
import { Observable, throwError } from 'rxjs';
import { map, tap, catchError, retry } from 'rxjs/operators';
import { Todo } from '../models/todo.model';

const API_URL = 'https://jsonplaceholder.typicode.com/todos';

// ── @Injectable ────────────────────────────────────────
// providedIn:'root' = one instance shared by whole app.
// No need to add to providers[] in AppModule.
@Injectable({ providedIn: 'root' })
export class TodoService {

  // ── DI: Angular provides HttpClient automatically ──
  constructor(private http: HttpClient) {}

  // ── GET all todos ──────────────────────────────────
  getAll(): Observable<Todo[]> {
    return this.http.get<Todo[]>(API_URL)
      .pipe(
        map(todos => todos.slice(0, 20)),   // keep first 20
        tap(todos => console.log('Loaded:', todos.length)),
        retry(2),                          // retry twice on failure
        catchError(this.handleError)
      );
  }

  // ── GET single todo by id ─────────────────────────
  getById(id: number): Observable<Todo> {
    return this.http.get<Todo>(`${API_URL}/${id}`)
      .pipe(
        catchError(this.handleError)
      );
  }

  // ── POST create new todo ───────────────────────────
  add(title: string): Observable<Todo> {
    const newTodo = { title, completed: false, userId: 1 };
    return this.http.post<Todo>(API_URL, newTodo)
      .pipe(
        tap(todo => console.log('Created:', todo)),
        catchError(this.handleError)
      );
  }

  // ── PUT update todo ────────────────────────────────
  update(todo: Todo): Observable<Todo> {
    return this.http.put<Todo>(`${API_URL}/${todo.id}`, todo)
      .pipe(catchError(this.handleError));
  }

  // ── DELETE todo ────────────────────────────────────
  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${API_URL}/${id}`)
      .pipe(catchError(this.handleError));
  }

  // ── Error handler ──────────────────────────────────
  private handleError(error: any) {
    console.error('API Error:', error.message);
    return throwError(() => new Error(error.message));
  }
}

import { Component, OnInit, OnDestroy } from '@angular/core';
import { ActivatedRoute, Router } from '@angular/router';
import { Subject, switchMap } from 'rxjs';
import { takeUntil } from 'rxjs/operators';
import { TodoService } from '../services/todo.service';
import { Todo } from '../models/todo.model';

@Component({
  selector:    'app-todo-detail',
  templateUrl: './todo-detail.component.html'
})
export class TodoDetailComponent implements OnInit, OnDestroy {

  // ── State ──────────────────────────────────────────
  todo:      Todo | null   = null;
  isLoading: boolean        = true;
  errorMsg:  string         = '';
  editTitle: string         = '';   ← bound to edit input

  private destroy$ = new Subject<void>();

  // ── DI: 3 services injected ───────────────────────
  constructor(
    private route:       ActivatedRoute,  // reads URL params
    private router:      Router,          // navigate
    private todoService: TodoService      // HTTP calls
  ) {}

  ngOnInit(): void {
    // ── switchMap: when :id param changes, cancel
    //    the previous request and start a new one ──
    this.route.paramMap
      .pipe(
        switchMap(params => {
          const id = Number(params.get('id'));
          return this.todoService.getById(id);
        }),
        takeUntil(this.destroy$)
      )
      .subscribe({
        next: (todo) => {
          this.todo      = todo;
          this.editTitle = todo.title;  // pre-fill edit field
          this.isLoading = false;
        },
        error: (err) => {
          this.errorMsg  = 'Todo not found';
          this.isLoading = false;
        }
      });
  }

  // ── Save edited todo ───────────────────────────────
  saveTodo(): void {
    if (!this.todo) return;
    const updated = {
      ...this.todo,
      title: this.editTitle   // take value from edit input
    };
    this.todoService.update(updated)
      .subscribe(saved => {
        this.todo = saved;
        alert('Saved!');
      });
  }

  // ── Go back to list ───────────────────────────────
  goBack(): void {
    this.router.navigate(['/']);
  }

  ngOnDestroy(): void {
    this.destroy$.next(); this.destroy$.complete();
  }
}

🏷️
ActivatedRoute
Read :id from the URL
🏷️ The House Number — When you navigate to /todos/5, the router writes "5" on the door. ActivatedRoute lets this component read that number. route.paramMap is an Observable that emits the current params — so if the URL changes, the component reacts automatically.
// Option 1: snapshot (one-time read) const id = this.route.snapshot.paramMap.get('id'); // Option 2: Observable (reacts to changes) this.route.paramMap.subscribe(params => { const id = params.get('id'); });
🔄
switchMap
Cancel old, start new
🔄 Changing the TV Channel — If the user navigates from /todos/5 to /todos/10 very quickly, switchMap cancels the request for todo #5 and starts a new one for #10. Without it, you might get todo #5's data arriving late and overwriting #10's data.
switchMap flattens an Observable of Observables. On each new emission, it unsubscribes from the previous inner Observable and subscribes to the new one.
this.route.paramMap.pipe( switchMap(params => { const id = Number(params.get('id')); return this.todoService.getById(id); // If a new id arrives before this completes, // this request is cancelled automatically! }) )

import { inject }     from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

// Modern functional guard (Angular 15+)
export const authGuard: CanActivateFn = () => {
  const router = inject(Router);
  const loggedIn = !!localStorage.getItem('token');

  if (loggedIn) return true;       // ✅ allow
  router.navigate(['/login']);     // ❌ redirect
  return false;
};

🔒 The Bouncer — A route guard is a bouncer at the door of a building. Before the router lets you into /admin, it asks the guard: "is this person allowed in?" If yes, they're in. If no, the guard redirects them to /login.
Guards are functions (or classes) that return true/false/UrlTree. canActivate runs before a route is activated. Modern Angular uses functional guards with inject().
export const authGuard: CanActivateFn = () => { const loggedIn = !!localStorage.getItem('token'); if (loggedIn) return true; inject(Router).navigate(['/login']); return false; }; // In routes: { path: 'admin', canActivate: [authGuard] }
